View это UI, то с чем взаимодействует непосредственно пользователь. С view должны работать дизайнеры.
В CodeBehind View ничего не должно содержаться. Вся логика и обработчики событий будут содержаться во view model.
Дизайнер спокойно разрабатывает во view ui, а программист может отдельно совершенно логику для нашей программы.
Такое разделение позволяет откреплять view и подключать другой, например открепляем view для desktop pc 
и подключаем view для отображения под mobile

Во view имеется только xaml разметка, по этому мы может отключить текущую view и подключить другую.

view может создаваться независимо, так как у нее нет CodeBehind. 
Дизайнеры могут разрабатывать view и потом просто передать программистам
